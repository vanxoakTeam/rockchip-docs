# SSH 网络远程登录指南

:::tip

本章节将指导您如何将 **RK3506-EVB** 连接至网络。设备支持 **百兆有线网络** 连接。


:::

  当用户当前使用环境没有USB转TTL串口模块时，可以在系统启动完成后（从上电后约等待15s左右），使用网络SSH登录到shell命令行进行调试；

## 1. 网络环境准备

  HD-RK3506-EVB评估套件标配2个以太网口，分布如下所示。

<img src={require('./images/eth.png').default} alt="image.png" style={{display: 'block', margin: '20px auto', maxWidth: '25%', borderRadius: '8px', boxShadow: '0 4px 8px rgba(0,0,0,0.1)'}}/>

  烧写万象奥科提供的出厂固件，未插入网线前，系统默认不会为网络节点分配IP，在插入网线后，系统会为ETH0分配默认的固定IP地址（192.168.1.10），用户可以使用该IP登录SSH后，使用ifconfig命令查看IP地址，如下所示。

```shell
ifconfig eth0
```

  如需要更改其他固定IP，可修改 /etc/network/interfaces 文件

```shell
# interface file auto-generated by buildroot

auto lo
iface lo inet loopback

auto eth0
iface eth0 inet static
address 192.168.1.10
gateway 192.168.1.1
netmask 255.255.255.0

auto eth1
iface eth1 inet static
address 192.168.1.12
gateway 192.168.1.1
netmask 255.255.255.0
```

参数说明：

-   `auto lo`：将 `lo` 标为**随系统启动自动拉起**的接口（`ifup -a` 会在引导阶段执行）。
-   `iface lo inet loopback`：定义逻辑接口 `lo`，地址族为 IPv4（`inet`），配置方法是 `loopback`（回环）。该方法无需额外选项。
-   `auto eth0`：把 `eth0` 也加入**开机自动启用**列表；若未写 `auto`（或 `allow-hotplug`），系统不会在启动时自动 `ifup` 它。
-   `iface eth0 inet static`：定义 `eth0` 的 **IPv4 静态地址** 配置段。`iface <名> <族> <方法>` 是 ifupdown 的核心句式。
-   `address 192.168.1.10`：为该接口分配的 **IPv4 地址**。
-   `gateway 192.168.1.1`：设置 **默认网关**；ifup 会据此添加默认路由。
-   `netmask 255.255.255.0`：**子网掩码**。
-   `auto eth1`：把 `eth0` 也加入**开机自动启用**列表；若未写 `auto`（或 `allow-hotplug`），系统不会在启动时自动 `ifup` 它。
-   `iface eth1 inet static`：定义 `eth0` 的 **IPv4 静态地址** 配置段。`iface <名> <族> <方法>` 是 ifupdown 的核心句式。
-   `address 192.168.1.12`：为该接口分配的 **IPv4 地址**。
-   `gateway 192.168.1.1`：设置 **默认网关**；ifup 会据此添加默认路由。
-   `netmask 255.255.255.0`：**子网掩码**。

  **HD-RK3506-EVB 两个网口都是固定IP。用户可以通过通过固定IP SSH访问开发板。**

  **（删减版网口通过ETH1访问，固定IP为192.168.1.12）**

  如果需要获取动态IP，可以通过以下方式。比如需要让ETH1动态获取IP，注释掉ETH1相关的静态IP设置，然后重启后即可自动获取IP。

```shell
# etc/network/interfaces文件，删除掉关于静态IP的描述，保存后重启

auto lo
iface lo inet loopback

auto eth0
iface eth0 inet static
address 192.168.1.10
gateway 192.168.1.1
netmask 255.255.255.0

#	auto eth1
#	iface eth1 inet static
#	address 192.168.1.12
#	gateway 192.168.1.1
#	netmask 255.255.255.0
```
